# Code and Architecture Review

## Overview

- Each product team that codes production applications will perform a code review with a product team outside of their product group to review their code bases, design patterns, security approach, data approach, and application architecture.
- Two of these code review sessions should happen per year.
- Briefback meeting notes are documented and shared within the product group Slack channel.

## Purpose
The purpose of the review is to:
  - Share knowledge
  - Foster learning
  - Provide feedback on where improvements can be made to software

## Suggested Discussion Format

1. Product Line Anchors pair up two product teams in different product lines to perform a review with one another.
2. Team A presents the following content for their product for feedback from Team B (1.5 hours):
   - Code bases
   - Design patterns
   - Security approach
   - Data approach
   - Application architecture
3. Team B presents the following content for their product for feedback from Team A (1.5 hours):
   - Code bases
   - Design patterns
   - Security approach
   - Data approach
   - Application architecture
4. Team A shares briefback notes to their product group Slack channel.
5. Team B shares briefback notes to their product group Slack channel.

## Guidelines

- Hold the reviews on separate days, i.e. don't perform reviews back-to-back.
- Pair teams with similar tech stacks.
