## What Is The Infrastructure Developer Role?

The *Infrastructure Developer* role is necessary for anyone who manages cloud infrastructure at the organization. ensures the data platforms run smoothly, securely and efficiently. The role is managed by system admins architects to make sure they build secured, scalable and reliable solutions in Google Cloud Portal (GCP).

Specifically, this role gives you access to manage GCP projects in Cloud Portal. It is required to:
 - Provision new GCP projects and managing existing projects. Please note that all projects need to be tied to an Enterprise Application Management System (EAMS) ID for creating a new project.
 - Creating or destroying environments, including sandbox projects.
 - Enabling or removing GCP native services.
 - Managing IAM roles and permissions for FIM groups and Vault/Applications Service Accounts.
 - Create Astronomer and Tekton namespaces for projects and control access to them CI/CD process.

## Who Needs To Have It?

The *Infrastructure Developer* role is only required for those responsible for administrating a product's infrastructure. The product anchor and product line anchor both need have the *Infrastructure Developer* role for the product's infrastructure. It is also useful product group anchor for added redundancy. Generally speaking, data engineers and AI engineers working on a product would not require this level of access.

## How Do You Request Access?

To request the role, you need to request access to the role for the specific project in [Access Management & Authorization](https://www.accessmgmt.company.com/CspsWeb/selectDataBegin.do?fromBookMark=true&entityName=ProductAppTechAccess&roleName=Infrastructure%20Developer). Search for the `Infrastructure Developer` role under the `ProductAppTechAccessDataService`, as shown in the first screenshot.

<img src="/images/analytics/anchor-guidance/access-management-infrastructure-dev-1.png" width="900" alt="Access Management Screenshot"/>

In order to submit the request, you will need to select the project's specific EAMS ID and then select all relevant environments. Once both have been selected, you can select the reviewer and submit the request.
