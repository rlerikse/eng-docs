## Clean Code Guidelines

Clean code enhances readability, maintainability and changeability of the code.

A few characteristics of clean code:
* Use meaningful names for variables, functions, objects and classes.
* Ensure right indentation of the code to enhance the readability of the code.
* Write testable code.
* Ensure full test coverage of the code.
* Reduce duplication in the code by modularizing repetitive logic into functions.
* Keep functions behaviors to the point and short.
* Avoid hard coding values in the code.
* Keep coupling between classes, components or modules to a minimum.
* Apply widely used design patterns to solve common problems in the code.
* Use tools like SonarQube for continuous inspection of code to check for quality and security vulnerabilities.
