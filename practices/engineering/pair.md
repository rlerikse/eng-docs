title: 'Pair Programming'
weight: 6
---
Pair programming is a collaborative software development methodology where code is developed by two software engineers
simultaneously. The primary benefit of pair programming is high code quality as code written by the driving software
engineer is reviewed by another (navigating software engineer) in real time. Software engineers in the pair regularly
switch between driving and navigating. *All teams in Platform+ Tech bias towards pair programming whenever possible.*

Pair programming secondary benefits:
* New software engineers on the team pairing with tenured software engineers will ramp-up on the codebase quickly
* Pairs don’t stay stuck on problems as long as they would in solo programming
* Pairs keep each other focused on solving the problem at hand and avoid building beyond scope
* Software engineers get real time feedback that helps them improve their own performance
* Design decisions are challenged
* More than one person has context into all domains of the code base
* Prevents the “hero mentality”

Pairing guidelines:
* Software engineers in the pair switch between the navigating and driving roles regularly.
* New engineers to the team may tend to navigate while they ramp-up but the tendency should be for them to work towards driving for an equal amount of time as their pair
* No keyboard hogging
* Ping pong pairing:
    * Engineer A drives the creation of the test, Engineer B drives the creation of the implementation
    * Engineers A and B switch roles for the next feature
