description: A metaphor for ensuring a proper testing strategy

---
The Software Testing Pyramid refers to grouping software tests into buckets of different granularity. It gives an idea 
of how many tests should be created within each group. Exploratory testing sits at the very top of the pyramid, 
followed by UI tests (automation) and then Integration and Unit tests. Unit and Integration tests are owned by software 
engineers while UI tests and Exploratory tests are owned by test engineers.
